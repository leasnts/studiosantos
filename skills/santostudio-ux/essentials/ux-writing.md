## Consistency Rules

### Terminology

Pick one term and stick with it:

| Don't Mix | Pick One |
|-----------|----------|
| Sign in / Log in / Login | Log in |
| Sign up / Register / Create account | Sign up |
| Settings / Preferences / Options | Settings |
| Delete / Remove / Trash | Delete |
| Cancel / Abort / Stop | Cancel |

### Capitalization

**Sentence case for most UI:**
```
âœ… "Create new project"
âŒ "Create New Project"

âœ… "Your account settings"
âŒ "Your Account Settings"
```

**Title case only for:**
- Product names
- Proper nouns
- Page titles (optional, be consistent)

### Punctuation

**No periods on:**
- Button labels
- Headlines
- List items (unless full sentences)
- Placeholder text

**Yes periods on:**
- Full sentences in descriptions
- Help text
- Error messages (if full sentences)

## Localization Considerations

### Write for Translation

- Avoid idioms ("break a leg", "piece of cake")
- Don't embed text in images
- Leave room for expansion (German is ~30% longer than English)
- Use complete sentences (fragments translate poorly)
- Avoid culturally specific references

### Variables in Strings

```
âŒ "You have " + count + " new messages"
âœ… "You have {count} new messages"

âŒ "Hello " + name
âœ… "Hello, {name}!"
```

## UX Writing Checklist

### Before Writing
- [ ] Who is the user in this moment?
- [ ] What do they need to know/do?
- [ ] What's their emotional state?
- [ ] What's the minimum words needed?

### Quality Check
- [ ] Would a human actually say this?
- [ ] Is it free of jargon?
- [ ] Is the action clear?
- [ ] Is it specific (not generic)?
- [ ] Does it match the brand voice?

### Consistency Check
- [ ] Same terms used throughout?
- [ ] Same capitalization pattern?
- [ ] Same punctuation rules?
- [ ] Same tone for similar situations?

### Content Reality Check
- [ ] No lorem ipsum anywhere?
- [ ] Names are diverse and realistic?
- [ ] Numbers are believable?
- [ ] Dates are in user's format?
- [ ] Content matches user's context?
